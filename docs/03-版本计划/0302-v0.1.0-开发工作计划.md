# 0302-v0.1.0-开发工作计划

## 一、文档目标

本计划用于承接当前已确认的原型设计与功能清单，指导 `v0.1.0` 首版开发落地。

本版本聚焦以下目标：
- 完成引导页与主画布页的基础开发
- 打通新建、浏览、连接、删除恢复四条核心闭环
- 完成摄像头、麦克风权限获取与失败兜底
- 建立本地存储、搜索定位、轻量模式等首版必要能力

## 三、开发原则

- **全中文**：文案、注释、计划、提示语、错误提示统一使用中文简体
- **先闭环再增强**：优先交付完整可跑通流程，再做高级动效和视觉细节
- **权限优先落地**：摄像头、麦克风权限是手势与语音入口的前置条件，必须优先验证
- **必须可降级**：每个核心动作都要有鼠标/触控兜底方案
- **本地优先**：首版不依赖后端，数据默认本地存储

## 七、摄像头与麦克风权限专项说明

### 7.1 摄像头权限

**目标**：
- 为手势识别提供视频流
- 在主画布右下角提供预览反馈

**开发要求**：
- [ ] 首次进入引导页时进行权限说明
- [ ] 用户进入主画布前可主动触发权限申请
- [ ] 权限允许后显示摄像头预览状态
- [ ] 权限拒绝后自动切换鼠标/触控主路径
- [ ] 若设备无摄像头，显示中文不可用提示

**验收标准**：
- [ ] 有权限时可正常显示预览
- [ ] 无权限时仍可继续使用产品核心功能

### 7.2 麦克风权限

**目标**：
- 为语音速记提供输入能力

**开发要求**：
- [ ] 在进入语音输入前进行权限申请
- [ ] 权限允许后进入录音/转写状态
- [ ] 权限拒绝后自动切换键盘输入
- [ ] 明确提示“不会保存原始录音，仅保留文本”

**验收标准**：
- [ ] 麦克风允许时可进入语音录入
- [ ] 麦克风拒绝时不阻塞卡片创建

## 九、风险与注意事项

### 9.1 高风险项

- 手势识别精度不稳定
- 浏览器权限行为差异
- 麦克风/语音能力在不同环境差异大
- 3D 表现与文本可读性冲突
- 动效过重影响性能

### 9.2 应对策略

- 优先保证鼠标/触控完整可用
- 权限失败时立即切换兜底流程
- 首版语音输入失败时无缝退回键盘输入
- 动效与粒子数量默认收敛
- 先完成 2D/2.5D 可用版本，再逐步增强空间感

# 褶宇宙 Fold-Space 原型设计文档

**文档用途**：供设计师、前端开发者、AI 原型工具生成可交互原型
**版本**：v1.1
**定位**：首版可落地原型，优先验证核心体验闭环而非一次性实现全部概念
**画布基准**：1440×900（桌面端），支持响应式缩放

---

## 一、产品概述

**核心隐喻**：用户站在一片虚空中，周围漂浮着发光的「时空薄片」（日记卡片），像星云一样分布。每张薄片代表一个记忆节点，通过光线连接形成「记忆星座」。

**首版交互原则**：
- **零点击优先**：核心流程可通过手势完成，但必须保留清晰的鼠标/触控兜底
- **体验先于炫技**：优先保证创建、浏览、连接、删除恢复四条闭环稳定成立
- **低认知负担**：首版减少手势数量与语义重叠，避免学习成本过高
- **视觉服务信息**：首版只保留最容易被理解的视觉编码，减少过载

**首版验证目标**：
- 用户能在 1 分钟内理解如何创建第一张薄片
- 用户能在无教程复读的情况下完成一次新建、一次连接、一次删除恢复
- 用户在低性能设备上依然能稳定完成核心流程

---

## 二、信息架构（页面地图）

```text
Landing（可选引导页）
    └── Main Canvas（主画布）── 唯一核心界面
            ├── 快速输入层（语音/极简标签）
            ├── 详情编辑舱（右侧滑出面板，深度编辑）
            ├── 废纸篓奇点（左下角回收站）
            ├── 手势引导层（半透明教学 UI）
            ├── 搜索虫洞层（搜索结果与定位）
            └── 系统状态栏（顶部极简信息）
```

**单页面应用（SPA）**：除首次权限申请外，所有操作在主画布完成，无页面跳转。

**视图模式定义**：
- **时间流视图**：默认视图，卡片按时间聚集分布
- **标签星云视图**：同标签卡片形成聚类光晕，突出主题分布
- **关联网络视图**：强化连线和节点关系，弱化时间轴存在感

**布局结构**：

```text
│     │ #工作 #焦虑     │    当前选中卡片显示关联锚点           │
│     ╰─────────────────╯                                      │
│               ╲                                              │
│                ╲────── ✦ ──────→  ╭──────────╮              │
│                                    │ 🧘 冥想    │              │
│                                    │ 其实是我   │              │
│                                    │ 太敏感...  │              │
│                                    ╰──────────╯              │
│                                                              │
│                                                              │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  [← 过去 ] ●━━●━━●━━●━━●━━●━━●━━●━━●━━●━━● [ 现在→]          │ ← 时间轴

```

**顶部栏分级**：
- **状态区**：`本地模式`、权限状态、性能状态
- **主操作区**：搜索、视图切换
- **次操作区**：帮助、导出、设置统一收纳进 `更多`

**3D 场景参数**：
- **背景**：深空渐变 `#0a0a12` → `#1a1b3a`
- **网格**：极淡透视网格，仅在拖拽、缩放、平移时增强显示
- **雾效**：轻量雾效，首版密度控制在不影响文本可读性的范围
- **光照**：环境光 + 柔和方向光，避免高对比导致文本发灰

**首版性能约束**：
- 默认视图稳定支持 `100` 张卡片
- 同屏高亮连线不超过 `30` 条
- 删除动画一次仅作用于单张卡片

### 3.2 时空薄片（Chrono-Fold Card）

**视觉规范**：

| 属性 | 规格 |
|------|------|
| **尺寸** | 宽度 280px，高度自适应（最小 120px，最大 320px，超出进入详情编辑） |
| **形状** | 圆角 16px，轻微不规则 |
| **材质** | 毛玻璃效果，透明度控制优先保证文字对比度 |
| **边框** | 1px 半透明边框，选中时增强 |
| **厚度** | 首版可视觉模拟厚度，不强依赖真实几何挤出 |

**首版视觉编码**：
- **位置**：表达时间
- **颜色**：表达情绪
- **连线/高亮**：表达关联

**暂缓的视觉编码**：
- 形状表达内容长度
- 光晕表达关联密度
- 大量常驻粒子表达环境氛围

**情绪色谱（正面）**：
- **冷静/夜晚**：`#1a1b3a`
- **积极/清晨**：`#ff6b35`
- **中性/白昼**：`rgba(255,255,255,0.92)`
- **低落/傍晚**：`#6b5b95`

**内容层级**：
```text
┌─────────────────────────┐
│ ○ 时间戳 · 情绪图标      │ ← 顶部 24px，12px
│                         │
│   主要内容文本...       │ ← 16px 正文，最大 5-6 行
│                         │
│ #标签 #标签              │ ← 底部标签区
└─────────────────────────┘
```

**状态变化**：
- **悬浮**：放大 1.03 倍，边框变亮
- **选中**：显示连接锚点与操作提示
- **待删除**：边缘发红光，显示“松手取消 / 保持确认”
- **私密模式**：内容模糊，仅显示色块和时间

### 3.3 输入模式 UI（Quick Input）

**触发方式**：
- 手势主路径：展开手掌 1.2s
- 鼠标兜底：点击空白处的 `+` 浮动入口或右键新建

**布局**：
```text
          ┌─────────────────────┐
          │   🎙️ 正在聆听...   │
          │                     │
          │   [波形可视化]      │
          │                     │
          │   "今天被老板..."   │ ← 实时转写
          │                     │
          │ [标签建议] [情绪色] │
          │ [保存]    [取消]    │
          └─────────────────────┘
```

**交互细节**：
- **录音中**：波形条根据音量高低跳动
- **转写中**：实时展示文本，允许删除最后一句
- **静默确认态**：转写结束后停留 3 秒，自动抽取标签与情绪，用户可快速调整
- **保存**：首版支持手势短握拳保存，也支持显式保存按钮
- **取消**：离开画面或点击取消后退出，不自动生成空卡片

**输入模式分层**：
- **默认**：语音速记
- **极简**：只选情绪和标签
- **深度**：进入详情编辑舱补全文本

---

### 3.4 删除与坍缩动画（Collapse Animation）

**设计原则**：保留宣泄感，但把“炫酷删除”与“不可逆销毁”拆开。

**触发**：
- 首次动作：长握拳或长按卡片进入待删除
- 确认结果：卡片坍缩进入废纸篓奇点，不在主画布直接不可逆删除

**四阶段动画**：

| 阶段 | 时长 | 视觉效果 |
|------|------|----------|
| **预警** | 0-0.5s | 红色光晕增强，卡片轻微震动 |
| **起皱** | 0.5-0.9s | 卡片中心向内凹陷，纸张褶皱增强 |
| **坍缩** | 0.9-1.2s | 卡片缩小并被吸向奇点方向 |
| **残留** | 1.2-1.6s | 少量碎片被吸入黑洞，画布恢复平静 |

**首版实现建议**：
- 粒子数控制在 `40-60`
- 优先用伪物理动画表现，不强依赖真实刚体
- 音效可选，默认关闭，用户主动开启

---

### 3.5 废纸篓奇点（Trash Singularity）

**位置**：左下角固定，圆形按钮 60×60px

**角色定义**：
- 它是“回收站”，不是立即销毁器
- 主画布删除的所有卡片先进入这里
- 真正不可逆删除只能在这里二次确认

**交互**：
- **点击/捏合**：展开奇点面板，展示最近删除卡片
- **恢复**：拖出或点击恢复，播放轻量反向展开动画
- **彻底粉碎**：在奇点面板内长按 2 秒并二次确认

**建议补充状态**：
- 空状态文案：“这里存放暂时放下的记忆”
- 非空状态显示数量和最近一次删除时间

---

### 3.6 手势引导层（Ghost Layer）

**首次使用或错误率偏高时显示**

**引导目标**：
- 让用户学会 `展开新建`
- 让用户学会 `捏合拖拽`
- 让用户学会 `长按删除并恢复`

**引导原则**：
- 不强制用户一次学完整套手势
- 每次只教一个动作
- 完成一个动作就立即给正向反馈并进入下一步

**动态提示（运行时）**：
- 检测到手：「🖐️ 已检测到手部」
- 可展开：「保持手掌展开，创建新记忆」
- 可拖拽：「捏住卡片并移动」
- 可删除：「保持动作确认放入废纸篓」

## 四、组件库（Component Library）

### 4.1 基础组件

**C1. 发光连线（Fold-Link）**
- 连接两张薄片
- 默认仅在选中态或网络视图增强显示
- 关系类型首版保留 2 种即可：`延续`、`跳转`

**C2. 时间褶皱轴（Time-Axis）**
- 底部水平条
- 默认用于定位时段和切换聚焦范围
- 首版避免承担过多 3D 导航职责

**C3. 搜索虫洞（Search Wormhole）**
- 关键词输入后高亮匹配卡片
- 相机飞向最佳匹配项
- 同时显示结果计数和“下一个结果”

### 4.2 状态组件

**S1. 加载态（Singularity Boot）**
- 中央旋转吸积盘动画
- 文案：「正在展开你的时空画布...」

**S2. 空态（Empty Void）**
- 中央悬浮提示：「张开手掌，或点击 +，从这里开始第一张薄片」
- 下方显示 2 个操作入口：`手势创建`、`点击创建`

**S3. 搜索空结果**
- 文案：「没有找到对应记忆，试试其他关键词」
- 提供“直接创建同名标签”快捷入口

## 五、交互规范（Interaction Spec）

### 5.1 首版核心手势集

| 手势 ID | 中文名 | 触发条件 | 主功能 | 备注 |
|---------|--------|----------|--------|------|
| **G-01** | 展开 | 五指张开并保持 1.2s | 新建薄片 | 主入口 |
| **G-02** | 捏合 | 拇指与食指靠近 | 拖拽薄片 | 含连接建立 |
| **G-03** | 长握拳 | 靠近目标卡片并保持 | 放入废纸篓 | 先删除到回收站 |
| **G-04** | 双手开合 | 双手距离变化 | 缩放视图 | 次优先级，可后补 |

**首版不作为必做项的手势**：
- 挥手撤销
- 紧急坍缩
- 双击捏合展开详情
- 语音输入期内的复杂切换手势

### 5.2 关键反馈映射

| 用户动作 | 界面反馈 | 延迟要求 |
|----------|----------|----------|
| 手进入画面 | 摄像头预览高亮一次 | < 100ms |
| 张开手掌 | 掌心出现倒计时环 | 实时 |
| 成功新建 | 输入舱浮出并聚焦 | < 100ms |
| 捏合卡片 | 卡片边缘变亮并跟手移动 | < 50ms |
| 删除确认 | 卡片变红并显示确认提示 | 实时 |
| 搜索命中 | 匹配卡片高亮并飞行定位 | < 300ms |

### 5.3 错误处理 UI

| 错误类型 | 反馈方式 |
|----------|----------|
| 手势置信度低 | 底部提示条：「请把手保持在画面中央」 |
| 误触删除 | 释放动作后立即取消，卡片恢复原状态 |
| 存储失败 | 顶部红色横幅 + 自动切换浏览器存储 |
| 麦克风拒绝 | 自动切换到键盘输入，保留当前文本框 |
| 性能不足 | 自动减少粒子和动态阴影，提示“已切换轻量模式” |

## 七、响应式适配

### 7.1 桌面端（Primary）
- 分辨率：1440×900 及以上
- 交互：手势优先，鼠标完整可用
- 布局：主画布 + 右侧编辑舱 + 底部时间轴

### 7.2 平板端
- 编辑舱改为底部弹层
- 取消复杂常驻粒子
- 手势仅保留新建与拖拽两类

### 7.3 移动端（降级模式）
- 默认不用摄像头手势
- 纯触控模式：
  - 长按空白处 = 新建
  - 拖拽卡片 = 排布
  - 拖拽到卡片上 = 建立连接
  - 长按卡片 = 删除到废纸篓
- 视觉降级为 2.5D，优先流畅度

## 八、原型检查清单

交付原型时需验证：

- [ ] 首次启动时，用户能在 30 秒内理解创建入口
- [ ] 新建流程支持手势与点击两条路径
- [ ] 连接建立后，关系表达清晰且不过度干扰阅读
- [ ] 删除后卡片进入废纸篓，可恢复，不会直接不可逆消失
- [ ] 搜索后，结果有高亮、有定位、有空结果处理
- [ ] 刷新页面后，内容、位置、标签保留
- [ ] 性能不足时能自动切轻量模式，核心流程不损坏
